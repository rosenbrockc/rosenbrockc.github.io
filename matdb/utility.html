matdb.utility.execute(args, folder, wait=True, nlines=100, venv=None, printerr=True, **kwargs)[source]¶

Executes the specified tuple that should include the command as first item and additional arguments afterward. See the documentation for subprocess.Popen for details.

Parameters:
  • args (list) – of str; first item should be command to execute; additional arguments following.
  • folder (str) – directory to switch into before executing the command.
  • wait (bool) – when True, block the current thread until execution completes; otherwise, returns immediately.
  • nlines (int) – by default, stdout and stderr are redirected to subprocess.PIPE. This is the maximum number of lines that will be returned for large outputs (so that memory doesn’t get overwhelmed by large outputs).
  • venv (str) – when not None, the name of a virtualenv to activate before running the command.
  • printerr (bool) – when True, if stderr is not empty, print the lines automatically.
  • kwargs (dict) – additional arguments that are passed directly to the subprocess.Popen constructor.
Returns:

with keys [‘process’, ‘stdout’, ‘stderr’], where ‘process’ is the instance of the subprocess that was created; ‘stdout’ and ‘stderr’ are only included if they were set to subprocess.PIPE.
Return type:

dict

Note

If the output from ‘stdout’ and ‘stderr’ are too large, only the first 100 lines will be returned. Use parameter nlines to control output size.
